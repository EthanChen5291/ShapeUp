# Security Policy

## Reporting A Vulnerability

Please report suspected security issues to [PLACEHOLDER_SECURITY_EMAIL].

Do not include personal data, biometric data, secrets, or exploit payloads beyond what is necessary to reproduce the issue.

We aim to acknowledge valid reports within [PLACEHOLDER_SECURITY_ACK_SLA] and provide updates until the issue is resolved.

## Scope

In scope:

- Authentication and authorization bypasses
- Data exposure involving account data, scan images, PLY files, or SPLAT files
- Payment or credit accounting abuse
- Server-side request forgery or unsafe file handling
- Prompt-injection paths that expose sensitive data or bypass validation

Out of scope until a formal program exists:

- Social engineering
- Denial-of-service testing beyond safe rate-limit validation
- Physical attacks
- Automated scanning that degrades service

This policy is a placeholder and needs human/security review before production publication.
