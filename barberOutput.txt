The core architectural decision: Barber Output is no longer "always call Gemini." A deterministic feasibility pass classifies every order first, and only two of five kinds ever reach the LLM:

no_change → no receipt. Notice says "no difference," with an optional "Maintenance — Same Cut" ticket (tighten fade, clean edges, same lengths) — which conveniently solves the "same as last time" barber pet peeve by handing them the actual spec.
buzz/bald → deterministic ticket, no LLM, no questions. "#1 all over" or "clippers no guard, razor finish."
growth_only → block notice ("a barber can't add length"), a grow-out plan with inches-to-go and a weeks estimate (anchored to the target spec's inches, since scene units have no physical scale), plus a "Shape-Up While Growing" edges-only ticket so the visit isn't wasted.
cut/mixed → up to 3 clarify chips, then Gemini, then validation with teeth.
